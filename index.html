Introduction

Our Backend project uses .NET 6 and follows a microservices architecture for modularity, scalability, and maintainability. using Entity Framework Core, CQRS, Repository Design Pattern, AWS SES for Messaging, Integration with PayStack, Azure Redis, Azure SQL Server, Azure Blob Storage, Docker, and more.

Project Structure

The project is structured as follows:

LIP Backend

  • src contains backend source.
    • api
      • WebCore
        • WebCore.API contains API project, controllers, middlewares
        • WebCore.Core contains core logic, Handlers, Commands, Query, Implements Repository Design Pattern
        • WebCore.Data contains models, entities, validators, and database context
        • docker-compose.yml and docker-compose.prod.yml define Docker configs.
  • README.md ReadMe file
  • .gitignore gitignore file

Architecture Overview

Our backend follow a CQRS architecture, with the Retailer backend utilizing Entity Framework Core. This architecture promotes separation of concerns and allows for efficient query and command handling.

Database

Uses SQL Server On project run, it creates a database and its tables. This centralizes data storage, facilitating data consistency and integrity.

Dockerization

Development Environment

Use docker-compose.yml for development. Run docker-compose -f docker-compose.yml -f docker-compose.override.yml up -d to start services.

Getting Started

  1. Clone the repository.
  2. Set up Docker and Docker Compose.
  3. Choose Docker config (docker-compose.yml for dev, docker-compose.prod.yml for prod).
  4. Run Docker commands to start services.
